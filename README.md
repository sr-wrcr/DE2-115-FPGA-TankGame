# DE2-115 双人对战坦克游戏（FPGA 实现）

本项目是一个基于 Altera DE2-115 开发板的**双人对战坦克游戏**。玩家通过 PS/2 键盘控制各自坦克移动、射击，游戏画面通过 VGA 接口输出到显示器，音效通过板载音频编解码器（WM8731）输出。游戏逻辑、碰撞检测、血量系统、障碍物、胜负判定等全部在 FPGA 上硬件实现。

## 🎮 游戏特性

- **双人对战**：两名玩家在同一台设备上进行实时对战
- **独立控制**：
  - 玩家1（红色坦克）：WASD 移动，空格射击
  - 玩家2（蓝色坦克）：方向键（↑↓←→）移动，小键盘数字0 射击
- **血量系统**：初始血量 5（或 15，由拨码开关选择），被击中后扣血并短暂无敌
- **障碍物**：可通过拨码开关启用/禁用，两种地图布局可选
- **动态难度**：拨码开关可调节游戏速度、子弹速度、射击冷却时间
- **胜负判定**：一方血量为 0 时游戏结束，VGA 屏幕显示 WIN / LOSE 字样
- **音效**：移动、射击、击中时播放不同音效（立体声区分左右玩家）
- **状态显示**：七段数码管显示双方血量和当前游戏参数

## 🖥️ 硬件平台

- **开发板**：Altera DE2-115 (Cyclone IV E)
- **显示**：VGA 接口（640×480 @ 60Hz）
- **输入**：PS/2 键盘接口
- **音频**：板载音频编解码器 WM8731（I2S 接口）
- **配置**：18 个拨码开关（SW[17:0]），4 个按键（KEY[3:0]）
- **状态显示**：8 个七段数码管


## 🔧 模块说明

### 1. `game.v`（顶层）
- 实例化 `keyboard`、`vga_keyboard`、`Synthesizer`
- 连接 VGA 输出引脚
- 将血量通过 `bin2bcd` + `seg` 显示到数码管
- 读取拨码开关配置并传递给游戏逻辑

### 2. `keyboard.v`
- PS/2 协议解码
- 支持按键按下/释放检测
- 输出：`p1_up/down/left/right/action`，`p2_up/down/left/right/action`

### 3. `vga_keyboard.v`（游戏核心）
- VGA 时序生成（640×480 @ 60Hz）
- 坦克移动、边界碰撞、障碍物碰撞
- 子弹管理（最多 8 发/玩家）
- 血量减少、无敌帧、游戏结束判定
- 像素级绘制：坦克、子弹、障碍物、中线、边框、胜负文字
- 输出音效触发信号（`p1_shoot_sound`、`p1_tank_move`、`p1_bullet_hit` 等）

### 4. `Synthesizer.v` + `tone_generator.v`
- 初始化 WM8731 音频芯片（I2C 配置）
- 生成 18.432MHz 音频主时钟
- `tone_generator` 根据触发信号生成不同频率和持续时间的音频波形
- `adio_codec` 将 16 位立体声数据通过 I2S 协议发送到 DAC

### 5. 音频数据流
```
音效触发 → tone_generator → 左/右声道数据 → adio_codec → I2S → WM8731 → 耳机/音箱
```

### 6. 七段数码管显示（从左到右）
- HEX7、HEX6：玩家1 血量（十位、个位）
- HEX5、HEX4：玩家2 血量（十位、个位）
- HEX3：游戏速度等级（SW[11:8]）
- HEX2：子弹速度等级（SW[7:5]）
- HEX1：射击冷却等级（SW[4:2]）
- HEX0：地图类型 + 障碍物开关（SW[1:0]）

## 🎛️ 拨码开关（SW）配置

| 开关范围 | 功能 | 说明 |
|----------|------|------|
| SW[17]   | 音频总开关 | 1 = 开启音效 |
| SW[16]   | 保留 | — |
| SW[15]   | 保留 | — |
| SW[14:13]| 保留 | — |
| SW[12]   | 血量模式 | 0 = 初始 5 血，1 = 初始 15 血 |
| SW[11:8] | 游戏速度 | 0~15，值越大移动越快（分频系数越小） |
| SW[7:5]  | 子弹速度 | 0~7，值越大子弹越快 |
| SW[4:2]  | 射击冷却 | 0~7，值越大冷却越短 |
| SW[1]    | 地图类型 | 0 = 地图A，1 = 地图B（障碍物布局不同） |
| SW[0]    | 障碍物启用 | 1 = 启用障碍物 |

## 🕹️ 按键说明

| 玩家 | 移动 | 射击 |
|------|------|------|
| 玩家1（红） | W / A / S / D | 空格键 |
| 玩家2（蓝） | ↑ / ↓ / ← / → | 小键盘 0 |

- **复位**：KEY[0]（按下复位游戏）
- **开始/暂停**：未实现（预留）

## 🚀 编译与下载

### 软件要求
- Quartus II 13.0 或更高版本（支持 Cyclone IV E）
- 建议使用 Quartus Prime 18.0 或更新版本

### 步骤
1. 打开 Quartus，新建工程，器件选择 `EP4CE115F29C7`（DE2-115）
2. 将所有 `.v` 文件添加到工程
3. 设置顶层模块为 `game`
4. 分配引脚（根据 DE2-115 用户手册）：
   - VGA 信号、PS/2 时钟/数据、音频 I2C/I2S、按键、拨码开关、数码管
   - 建议直接导入提供的 `.qsf` 引脚分配文件（如有）
5. 综合、编译、生成 `.sof` 文件
6. 通过 JTAG 下载到 DE2-115
7. 连接 VGA 显示器、PS/2 键盘、耳机/音箱，上电即可开始游戏

> 注：若没有现成的引脚分配文件，请参考 `DE2_115_pin_assignments.csv`（需自行根据手册编写）。

## 📝 注意事项

- 游戏刷新率为 60Hz，所有逻辑在 25MHz 像素时钟下同步
- 子弹移动和坦克移动使用不同的分频计数器，可通过拨码开关独立调节速度
- 无敌时间约为 2 秒（50MHz 时钟下计数 100,000,000 次）
- 音频主时钟 18.432MHz 由 PLL 从 50MHz 产生，需确保 PLL 锁定后再开始音频传输
- 游戏结束后需要按 KEY[0] 复位才能重新开始

## 🧪 已知限制

- 不支持暂停功能
- 子弹和坦克碰撞判定为矩形框，精度一般
- 障碍物位置固定，不可动态生成
- 最多支持 8 发子弹同时存在，超过上限则无法射击直到子弹消失
